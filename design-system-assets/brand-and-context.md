# Global Bali Home — Brand & Design System Context

## Company

**Global Bali Home** — инвестиционный девелоперский холдинг на Бали (Индонезия). Премиальная недвижимость для иностранных инвесторов: виллы, эстейты, виллидж-проекты под управлением.

**Не**: «девелопер вилл», «турагент», «риелтор». **Да**: холдинг, инвестиции, легальная схема владения для иностранцев.

## Аудитория

- ~50% русскоязычные, ~50% англоязычные (сайт двуязычный, EN/RU параллельно)
- Возраст 35–55, осознанные инвесторы, не первый объект
- Ищут: легальность, понятный ROI, доверие к девелоперу, premium-уровень

## Тон бренда

- Спокойный, уверенный, премиум
- Без хайпа, без «уникальных предложений только сегодня»
- Факты + цифры + социальное доказательство (команда, партнёры, проекты)
- Эстетика: природа Бали (рисовые террасы, Убуд, тёплый свет), архитектурный минимализм

## Визуальный язык

- **Тёмная тема** — публичный сайт. Тёплая палитра (бежевый текст на тёмно-зелёно-чёрном фоне), оливковый акцент `#6B8F4E`
- **Шрифты**: Playfair Display (заголовки, классический serif с премиум-ощущением) + Montserrat (body, чистый sans)
- **Фотография**: природа Бали, интерьеры вилл, рисовые поля, тёплый свет золотого часа
- **Иконки**: stroke-only, тонкие, минималистичные (stroke-width: 1.5)
- **Композиция**: много воздуха, крупные hero-картинки, сдержанный CTA

## Структура сайта

18 страниц = 9 EN (корень) + 9 RU (`ru/` папка):
- Home — обзор + социальное доказательство
- Projects — каталог
- Project pages: Serenity Villas, Serenity Estates, Serenity Village
- Services — что делаем
- About — команда, миссия, партнёры
- Gallery — фото
- Contact

## Технический стек

- Чистый HTML/CSS/JS, без фреймворков, без сборщика
- Деплой через Vercel (push в master = live)
- Все интерактивы в `js/main.js` (попапы, scroll reveals, галерея, формы, калькулятор)
- Данные централизованы в `data/site-data.js`

## Две независимые дизайн-системы

**ВАЖНО**: в проекте две дизайн-системы, их НЕ смешивать.

1. **Публичный сайт** (`css/style.css`) — тёмная премиум-тема для маркетингового сайта. Описана в `palette-and-tokens.md`. **Это то, что нужно расширять.**

2. **Админка** (`admin/`) — отдельная светлая система с токенами `--sp-*`, своими компонентами (.form-group, .form-grid, editor-section). Сюда **не лезть** — у неё свой брендбук.

## Что хочется улучшить / куда расти

- Единые компоненты карточек проектов (сейчас вариации между страницами)
- Консистентность hero-секций между Project pages
- Чёткая система spacing-токенов (сейчас только `--section-padding`/`--side-padding`, не хватает шкалы как в админке `--sp-*`)
- Расширить scale цветов (нет промежуточных оттенков accent)
- Добавить токены для shadow/elevation (сейчас тени хардкодом)
