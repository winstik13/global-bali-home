# CLAUDE.md

Инструкции для Claude Code при работе с этим репозиторием.

## Проект

**Global Bali Home** — маркетинговый сайт премиального девелопера недвижимости на Бали. Чистый HTML/CSS/JS, без фреймворков и сборщиков.

## Архитектура

- **27 страниц**: 9 EN (корень) + 9 RU (`ru/`) + 9 ID (`id/`). Актуальный список — через `Glob *.html`, `ru/*.html`, `id/*.html`.
- `css/style.css` — дизайн-система публичного сайта (тёмная тема, responsive 1024/768px)
- `css/reset.css` — CSS reset
- `js/main.js` — вся интерактивность (скролл, меню, анимации, галерея, формы, попапы, калькулятор)
- `data/site-data.js` — централизованные данные. **Правятся через админку**, не руками.
- `gallery-data.js` — автогенерируется через `generate-gallery.ps1`
- `D:\global-bali-docs\Base info.md` — справочник актуальных данных проектов (вынесен из проекта). **Сначала обновлять там**, потом синхронизировать в HTML.
- `admin/` — админка, **свои правила** в [`admin/CLAUDE.md`](admin/CLAUDE.md)

### JS-инжектируемые компоненты (DOM создаётся в main.js)

- **Квиз-попап** — квалификация лида. Триггер: `[data-quiz]` или `.cta-section .btn--primary:not([data-tour])`
- **Tour-попап** — запись на показ. Триггер: `[data-tour]` или `[data-tour="Project Name"]` (последний пропускает шаг выбора проекта)
- **Exit intent попап** — `mouseleave` + задержка. Настройки из `SITE_DATA.exitPopup`
- **Sticky CTA бар** — нижняя панель на мобильных. Страницы проектов → tour, остальные → quiz

## Workflow

- **Нет build-шага**. Открыть `.html` в браузере для теста.
- **Push = деплой** (~1-2 мин, ветка `master` → https://winstik13.github.io/global-bali-home/). **Всегда подтверждать push перед выполнением.**
- Актуальные данные проектов сначала обновлять в `D:\global-bali-docs\Base info.md`, потом в HTML.

## Обязательные правила кода

### Мультиязычность и пути

- При изменении шапки, подвала, WhatsApp-кнопки, глобальных компонентов — **реплицировать на все 27 файлов** (9 EN + 9 RU + 9 ID)
- В `ru/`, `id/`: префикс `../` для CSS, JS, изображений, видео. Навигация — **без** префикса (`index.html`, не `../index.html`)
- Языковой переключатель: `../` для EN, `../ru/`, `../id/` для остальных
- Внешние ссылки: **всегда** `target="_blank" rel="noopener noreferrer"`

### HTML и SEO

- Изображения ниже fold: `loading="lazy"`
- Каждая страница: `<title>`, meta description, OG, Twitter Card, canonical, favicon. Проекты и контакты — `JSON-LD`.
- Порядок навигации: **Home → Projects → Services → About → Gallery → Contact** + CTA + язык

### CSS / дизайн-система публичного сайта

- **БЭМ**: `block__element--modifier`
- **CSS-переменные**: цвета, шрифты, отступы — **только** через `var(--...)`, без хардкода
- **Токены публичного сайта**: `--section-padding`, `--side-padding`. Спейсинг-токенов `--sp-*` **нет** (они есть только в админке)
- **Шрифты**: `--font-heading` (Playfair Display) для заголовков, `--font-body` (Montserrat) для body. Размеры через `clamp()`.
- **Чередование фона секций**: `.bg-alt` для альтернативных, не допускать слияния
- **SVG-иконки**: `stroke: var(--color-accent); stroke-width: 1.5; fill: none`
- **Scroll-анимации**: классы `.reveal`, `.reveal-left`, `.reveal-right`, `.reveal-stagger` + `IntersectionObserver` добавляет `.visible`
- **Счётчик чисел**: атрибут `data-counter`
- **Публичный сайт и админка — разные дизайн-системы**. Не смешивать. См. [`admin/CLAUDE.md`](admin/CLAUDE.md)

### CSS gotchas (из реальных багов)

- **Hover-scale ломает дочерние**: если элемент содержит absolute-позиционированные дочерние (тултипы, бейджи), **НЕ использовать** `transform: scale()` в `:hover`. Дочерние масштабируются вместе с родителем, ломая предсказуемость размеров. Использовать `box-shadow` / ring вместо scale.
- **`.reveal.visible` перебивает transform**: класс `.reveal` ставит `transform: translateY(...)`. Если нужно `rotate/scale` на том же элементе — вынести в дочерний элемент, иначе reveal через specificity перебьёт.
- **Inline `<span>` игнорирует `width`/`min-height`**: если нужна фиксированная коробка — `display: block` или `inline-block`.
- **Тултипы/попапы внутри `<button>`**: явно задавать `font-family`, не полагаться на `inherit` — браузеры применяют системный font-reset к кнопкам, и он может протекать.

## Skills vs Subagents

| Задача | Инструмент |
|--------|-----------|
| Итеративно оценить один элемент/секцию, с которым работаем | `/audit` skill (бывший `/review`) |
| Прогнать весь сайт целиком | subagent (`full-audit`, `pre-deploy`, `responsive-audit`, `marketing-copy`) |
| Валидировать SEO-разметку | subagent `seo-check` |
| Проверить консистентность переводов EN/RU/ID | subagent `i18n-audit` |
| Синхронизировать HTML EN → RU/ID после структурных изменений | subagent `page-sync` |

**Пересечения:**
- `marketing-copy` subagent ⇄ `/audit --marketing` skill — subagent для массовой проверки нескольких страниц, skill для конкретной секции
- `responsive-audit` subagent ⇄ `/audit --ux` skill — subagent прогоняет весь сайт, skill проверяет один компонент

**Правило:** если задача узкая и итеративная — skill; если широкая и «прогнать целиком» — subagent.

**Важно:** subagents запускаются **только по явному запросу** пользователя. Не вызывать самостоятельно.

## Авто-роутинг для `/audit`

При запросе на оценку/анализ/проверку без явной команды — определить намерение по триггерным словам и вызвать `/audit` с правильными флагами.

### Триггеры → флаги

| Триггерные слова / фразы | Флаг |
|--------------------------|------|
| «оцени текст», «копирайт», «CTA», «это продаёт», «tone of voice», «офер», «заголовок» | `--marketing` |
| «удобно ли», «понятно ли», «тач-зоны», «мобильная версия», «accessibility», «навигация» | `--ux` |
| «дизайн-система», «БЭМ», «хардкод», «переменные», «консистентно», «цвета» | `--ui` |
| «шрифты», «размеры текста», «иерархия заголовков», «типографика», «читается ли» | `--typo` |
| «отступы», «spacing», «padding», «margin», «вертикальный ритм», «выравнивание» | `--spacing` |
| «проверь визуально», «глянь стилистику», «оцени секцию», «посмотри как выглядит», «что думаешь» | (без флагов = всё) |
| «быстро глянь», «критичное», «перед деплоем», «что срочно фиксить» | `--quick` |

### Confirmation gate

- **Высокая уверенность (>80%)** → запускать сразу, в начале отчёта одной строкой: *«Интерпретирую как `/audit [flags]`. Если не то — скажи.»*
- **Средняя (50-80%)** → один уточняющий вопрос: *«Понял это как X. Делаем X или полный аудит?»*
- **Низкая (<50%)** → спросить варианты списком (1. Маркетинг / 2. Визуал / 3. UX / 4. Всё)

### Контекстные подсказки

Если запрос неоднозначный — смотреть:
1. **Какие файлы сейчас редактируем**: `*.css` → визуальные флаги; `*.html` → `--marketing --ux`; `admin/*` → админская дизайн-система
2. **Последние 3-5 сообщений**: чинили CSS → визуальные; меняли тексты → `--marketing`; добавляли секцию → полный аудит
3. **Скриншот в сообщении**: секция → полный аудит; форма/админка → `--ui --spacing`; текст/CTA → `--marketing`

### НЕ запускать `/audit`

Триггеры «почему так», «как работает», «поясни код», «почини баг», «ошибка в», «не работает» → отвечать инлайн без скилла.

### Обучение на ошибках

Если пользователь исправил роутинг («не то, мне нужно было только X») — зафиксировать в memory как feedback: «при словах [фраза] → `/audit --X`». В следующий раз применять.
