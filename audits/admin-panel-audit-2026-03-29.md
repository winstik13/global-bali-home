# Полный аудит админ-панели Global Bali Home
**Дата:** 2026-03-29 21:25
**Файлы:** admin/index.html, admin/admin.css, admin/admin.js (2849 строк)
**Архитектура:** Firebase Auth + GitHub API, без бэкенда, деплой через git push

## 1. Обзор функциональности

### Экраны (3 состояния приложения)

| Экран | ID | Описание |
|---|---|---|
| Логин | `#login-screen` | Email + пароль через Firebase Auth. Переключатель EN/RU в правом углу. |
| GitHub PAT | `#pat-screen` | Ввод Personal Access Token. Чекбокс "Запомнить на устройстве" (localStorage vs sessionStorage). |
| Основное приложение | `#admin-app` | Показывается после двойной аутентификации (Firebase + PAT). |

### Шапка (header)

- Логотип + название "Admin Panel / Панель управления"
- Виджет курса USD/IDR — обновляется из `open.er-api.com` каждый час
- Переключатель языка EN/RU
- Email залогиненного пользователя
- Кнопка "Sign Out / Выйти"

### Навигация (7 вкладок)

| Вкладка | data-tab | Что делает |
|---|---|---|
| Dashboard / Обзор | `dashboard` | Сводка по всем проектам + последние коммиты + курс + контакты + PDF гид |
| Projects / Проекты | `projects` | Редактор юнитов, типов, доступности, hero-статистики, showcase-текстов |
| SEO | `seo` | Редактор title/description/OG для 6 страниц × 3 языка + live-превью Google SERP и OG |
| Gallery / Галерея | `gallery` | Загрузка/удаление/сортировка фото по проекту. Drag & drop. Авторесайз до 1920px. |
| FAQ | `faq` | Редактор FAQ на 3 языках, сортировка, добавление/удаление |
| Testimonials / Отзывы | `testimonials` | Редактор отзывов: имя, роль, текст (3 языка), рейтинг звёзды 1–5, сортировка |
| Colors / Цвета | `colors` | Редактор 6 CSS-переменных (фоны, текст, акцент) + live-превью в самой админке |

### Компоненты Dashboard (внутри вкладки)

1. **Сводная строка** — Total Units, Sold/Booked, Available, Overall %, Est. Revenue
2. **Карточки проектов** — для каждого проекта: название, статус-бейдж, 4 метрики (sold/total, left, от цены, выручка), прогресс-бар, breakdown по статусам юнитов, кнопки "Редактировать" + "На сайте"
3. **Recent Changes** — последние 5 коммитов `data/projects-data.js` через GitHub API
4. **Курс валют (USD→IDR)** — ручной ввод или авто (API) + кнопка сохранить
5. **Контактная информация** — телефон, WhatsApp, email, адрес (EN/RU/ID) с валидацией
6. **PDF Гид по инвестициям** — загрузка PDF в репозиторий

### Project Editor (внутри вкладки Projects)

- Динамические вкладки по проектам + кнопка "+ New Project" (модальное окно)
- Таблица юнитов: ID, тип, этажи, площадь, участок, бейдж, статус, цена, удалить
- Таблица типов юнитов (для Village): тип, этажи, площадь, участок, кол-во, цена
- Блок доступности: sold / total (авторасчёт из статусов)
- Hero Stats (EN/RU/ID): числа + подписи, 4 стата
- Showcase Card (EN/RU/ID): цена + описание
- Кнопки: "Publish Changes" + "Generate Detail Pages"

### Механизм публикации

Все изменения сохраняются через GitHub Contents API (`PUT /repos/{owner}/{repo}/contents/{path}`):
- `data/projects-data.js` — данные проектов
- `data/faq-data.js` — FAQ
- `data/testimonials-data.js` — отзывы
- `data/site-data.js` — курс, контакты, цвета, PDF-мета
- `gallery-data.js` — манифест галереи
- HTML-файлы страниц — SEO и генерация детальных страниц

## 3. Маркетинговый анализ

### Сильные стороны (что работает хорошо)

1. **Dashboard с реальными KPI** — сводка юнитов (Total/Sold/Available + выручка) позволяет за 5 секунд оценить состояние продаж без входа в GitHub или таблицы.

2. **Прямое редактирование цен и статусов юнитов** — ключевое для рынка недвижимости. Смена "available" → "sold" для конкретного юнита обновляется на всём сайте после публикации. Это быстрее, чем редактировать HTML вручную.

3. **SEO с live-превью** — SERP-превью в стиле Google и OG-превью в стиле Facebook позволяют нетехническому пользователю понять, как выглядит страница при поиске, без знания HTML.

4. **Курс USD/IDR с авторежимом** — критично для Индонезии, где IDR обязателен. Авторежим снимает рутину ежедневного обновления.

5. **Загрузка фото с авторесайзом** — изображения автоматически масштабируются до 1920px с качеством 0.8 перед загрузкой в репозиторий. Это защищает от загрузки 8MB RAW-файлов с телефона.

6. **Генерация детальных страниц** — позволяет запустить новый проект без программиста: заполнить форму → нажать Generate → получить 3 HTML-страницы (EN/RU/ID).

## Технические долги (вне рамок аудита, но стоит зафиксировать)

- **GitHub PAT в localStorage** — если устройство общее или скомпрометировано, токен с `contents:write` scope доступен любому JS на странице. Рассмотреть хранение только в памяти (sessionStorage — уже есть опция, но по умолчанию используется localStorage).
- **Firebase config публичен** (строки 500–503) — `apiKey` в клиентском коде. Это ожидаемо для Firebase Auth, но важно настроить правила безопасности Firebase Console (допустить только конкретные email-адреса).
- **`btoa(unescape(encodeURIComponent(content)))`** (строка 1675) — устаревший паттерн кодирования UTF-8 в Base64. Современный вариант: `btoa(String.fromCharCode(...new TextEncoder().encode(content)))`.
- **Нет rate limiting для операций записи** — пользователь может нажать "Publish" несколько раз подряд, исчерпав GitHub API лимит (5000 запросов/час). Кнопка блокируется через `btnLoading()`, но только на время одного запроса.
