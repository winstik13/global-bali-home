## ОТЛОЖЕННЫЕ ПУНКТЫ

### SEO

**[SEO-1] Нет страницы Privacy Policy** — ⏸ ОТЛОЖЕНО
- **Суть:** Формы собирают email, телефон, имя. Чекбоксы consent ссылаются на "Privacy Policy", но страницы нет.
- **Причина:** Нужен юридический текст. Создание страницы на 3 языках — отдельная задача с привлечением юриста или шаблона.

**[SEO-5] Alt-тексты изображений слишком общие** — ⏸ ОТЛОЖЕНО
- **Суть:** `alt="Aerial view"`, `alt="Living room"` — без упоминания проекта или Bali. Слабый SEO-сигнал.
- **Причина:** Много файлов (27 HTML), механическая работа. Лучше делать отдельным проходом.

**[SEO-7] Нет VideoObject schema для hero-видео** — ⏸ ОТЛОЖЕНО
- **Суть:** Hero-секции используют `<video>`, но нет VideoObject structured data.
- **Причина:** Минимальное влияние — Google редко индексирует фоновые декоративные видео.

### CSS

**[CSS-2] Нет print-стилей** — ⏸ ОТЛОЖЕНО
- **Суть:** При печати страницы остаются навигация, floating кнопки, анимации.
- **Причина:** Целевая аудитория (инвесторы) не печатает сайт. Низкий приоритет.

**[CSS-3] Нет breakpoint для маленьких экранов (375px, 480px)** — ⏸ ОТЛОЖЕНО
- **Суть:** Текущие breakpoints: 1024px, 900px, 768px. На iPhone SE (375px) могут быть проблемы с layout.
- **Причина:** Нужен визуальный тест на реальных устройствах для определения конкретных проблем.

### JavaScript

**[JS-1] Квиз и exit popup: нет focus trap** — ⏸ ОТЛОЖЕНО
- **Суть:** При открытом модале Tab уводит фокус на элементы под ним. Скринридеры не ограничены модалом.
- **Причина:** Требует написания утилиты focus trap и тестирования со скринридерами. Делать вместе с A11Y-1.

**[JS-3] Exit popup не показывается на мобильных** — ⏸ ОТЛОЖЕНО
- **Суть:** Триггер по `mouseleave` — на телефонах нет мыши, popup никогда не покажется.
- **Причина:** Time-based триггер нужно тестировать на реальных устройствах, чтобы не раздражать пользователей.

**[JS-4] В лайтбоксе нельзя свайпать фото на телефоне** — ⏸ ОТЛОЖЕНО
- **Суть:** Листать фото можно стрелками и кнопками, но свайп пальцем не работает.
- **Причина:** ~30-40 строк touch events, нужно тестировать на реальных устройствах.

### Маркетинг

**[MKT-1] Нет Privacy Policy и Terms of Service** — ⏸ ОТЛОЖЕНО
- **Суть:** Формы собирают персональные данные, но юридических страниц нет. Совпадает с SEO-1.
- **Причина:** Нужен юридический контент, не код.

**[MKT-2] Нет cross-linking между проектами** — ⏸ ОТЛОЖЕНО
- **Суть:** На странице проекта нет блока "Другие проекты". Если проект не подходит — пользователь уходит.
- **Причина:** Нужно дизайн-решение (layout карточек, контент). Отдельная фича.

**[MKT-3] Нет urgency-элементов на presale Village** — ⏸ ОТЛОЖЕНО
- **Суть:** Serenity Village в presale, но нет визуальных индикаторов срочности (countdown, early bird).
- **Причина:** Нужно бизнес-решение — countdown к какой дате? Есть ли реальная дата окончания presale?

### Performance

**[PERF-1] Множество scroll-обработчиков** — ⏸ ОТЛОЖЕНО
- **Суть:** 3 отдельных scroll handler с rAF throttle (header, parallax, sticky CTA).
- **Причина:** Каждый handler работает корректно с rAF. Объединение усложнит код без заметного выигрыша. Параллакс уже отключён на мобильных (JS-5).

### Accessibility

**[A11Y-1] Мобильное меню: нет focus trap** — ⏸ ОТЛОЖЕНО
- **Суть:** При открытии бургер-меню Tab уводит фокус за пределы навигации.
- **Причина:** Делать вместе с JS-1 (общая утилита focus trap для всех модалов).

**[A11Y-2] Gallery alt-тексты генерируются из имён файлов** — ⏸ ОТЛОЖЕНО
- **Суть:** `img.alt = src.split('/').pop().replace(...)` даёт "villa a1 exterior" вместо описательного текста.
- **Причина:** Нужно добавить поле alt в GALLERY_DATA и заполнить для каждого изображения.
